    SWBClient Reference¶

    swb.api.SWBClient ¶

    Client for interacting with the SWB SRU API.

    The SWB (Südwestdeutscher Bibliotheksverbund) provides an SRU (Search/Retrieve via URL) interface for searching library catalogs.

    Attributes:

    Name Type Description
    base_url

    Base URL for the SRU endpoint
    timeout

    Request timeout in seconds
    session

    Requests session for connection pooling

    explain() ¶

    Retrieve server capabilities and configuration (SRU explain operation).

    The explain operation returns information about the server including: - Server and database information - Available search indices - Supported record schemas/formats - Configuration details

    Returns:

    Type Description
    ExplainResponse

    ExplainResponse containing server capabilities.

    Raises:

    Type Description
    RequestException

    If the API request fails.
    ValueError

    If the response cannot be parsed.
    Example

    client = SWBClient() info = client.explain() print(f"Database: {info.database_info.title}") print(f"Available indices: {len(info.indices)}") for schema in info.schemas: ... print(f" - {schema.name}")

    scan(scan_clause, response_position=1, maximum_terms=20) ¶

    Scan an index for terms (for auto-completion and browsing).

    The scan operation allows browsing index terms, which is useful for: - Auto-completion in search interfaces - Exploring available terms before searching - Finding correct spellings

    Parameters:

    Name Type Description Default
    scan_clause str

    CQL scan clause (e.g., "pica.per=Goe" to find authors starting with "Goe").

    		 required
    response_position int

    Position in the term list to start from (1-based).

    		 1
    maximum_terms int

    Maximum number of terms to return.

    		 20

    Returns:

    Type Description
    ScanResponse

    ScanResponse containing the list of terms.

    Raises:

    Type Description
    RequestException

    If the API request fails.
    ValueError

    If scan_clause is empty, if response_position < 1, if maximum_terms < 1, or if the response cannot be parsed.
    Example

    client = SWBClient() response = client.scan("pica.per=Goe", maximum_terms=10) for term in response.terms: ... print(f"{term.value}: {term.number_of_records} records")

    search(query, record_format=RecordFormat.MARCXML, start_record=1, maximum_records=10, index=None, sort_by=None, sort_order=SortOrder.DESCENDING, record_packing='xml', facets=None, facet_limit=10, sru_version=None) ¶

    Search the SWB catalog using CQL query syntax.

    Parameters:

    Name Type Description Default
    query str

    Search query. Can be a simple keyword or full CQL query.

    		 required
    record_format RecordFormat

    Desired format for the results.

    		 MARCXML
    start_record int

    Position of first record to return (1-based).

    		 1
    maximum_records int

    Maximum number of records to return.

    		 10
    index SearchIndex | None

    Search index to use. If provided, query is treated as a simple keyword search in that index.

    		 None
    sort_by SortBy | None

    Sort results by relevance, year, author, or title.

    		 None
    sort_order SortOrder

    Sort order (ascending or descending). Default is descending.

    		 DESCENDING
    record_packing str

    How records are packed in the response. Valid values: "xml" (default) - Records embedded as XML "string" - Records escaped as strings

    		 'xml'
    facets list[str] | None

    List of facet fields to request (requires SRU 2.0). Examples: ["year", "author", "subject"]

    		 None
    facet_limit int

    Maximum number of facet values per facet field. Default is 10.

    		 10
    sru_version str | None

    SRU version to use. If facets are requested, defaults to "2.0", otherwise uses the client's default version.

    		 None

    Returns:

    Type Description
    SearchResponse

    SearchResponse containing the search results and facets (if requested).

    Raises:

    Type Description
    RequestException

    If the API request fails.
    ValueError

    If the response cannot be parsed, parameters are invalid, or recordPacking is invalid.
    Example

    client = SWBClient() results = client.search( ... "Python", ... index=SearchIndex.TITLE, ... sort_by=SortBy.YEAR, ... sort_order=SortOrder.DESCENDING, ... facets=["year", "author"], ... facet_limit=20 ... ) print(f"Found {results.total_results} results") if results.facets: ... for facet in results.facets: ... print(f"{facet.name}: {len(facet.values)} values")

    search_related(ppn, relation_type, record_type=RecordType.BIBLIOGRAPHIC, record_format=RecordFormat.MARCXML, start_record=1, maximum_records=10, sort_by=None, sort_order=SortOrder.DESCENDING, record_packing='xml') ¶

    Search for records related to a specific publication (band/linking search).

    This method is useful for finding related publications in multi-volume works, series, or hierarchical bibliographic relationships. It uses K10plus-specific search attributes to retrieve linked records.

    Parameters:

    Name Type Description Default
    ppn str

    PPN (PICA Production Number) of the parent record to find related records for.

    		 required
    relation_type RelationType

    Type of relationship to search for (e.g., parent, child, family, related, thesaurus).

    		 required
    record_type RecordType

    Type of records to retrieve (bibliographic or authority). Default is bibliographic records.

    		 BIBLIOGRAPHIC
    record_format RecordFormat

    Desired format for the results.

    		 MARCXML
    start_record int

    Position of first record to return (1-based).

    		 1
    maximum_records int

    Maximum number of records to return.

    		 10
    sort_by SortBy | None

    Sort results by relevance, year, author, or title.

    		 None
    sort_order SortOrder

    Sort order (ascending or descending). Default is descending.

    		 DESCENDING
    record_packing str

    How records are packed ("xml" or "string"). Default is "xml".

    		 'xml'

    Returns:

    Type Description
    SearchResponse

    SearchResponse containing the related records.

    Raises:

    Type Description
    RequestException

    If the API request fails.
    ValueError

    If ppn is empty, if start_record < 1, if maximum_records < 1, or if the response cannot be parsed.
    Example

    client = SWBClient()

    Find all child records (volumes) of a multi-volume work¶

    results = client.search_related( ... ppn="267838395", ... relation_type=RelationType.CHILD, ... maximum_records=20 ... ) print(f"Found {results.total_results} related records")
